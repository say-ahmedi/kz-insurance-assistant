# ИИ-ассистент: Страховое законодательство РК

RAG-ассистент для внутреннего использования в страховой компании.
Отвечает на вопросы сотрудников по законам Республики Казахстан о
страховании на основании загруженных текстов нормативных актов.

## Архитектура

```
┌────────────┐
│  laws/     │  .docx файлы законов РК (можно дополнять)
│ *.docx     │
└─────┬──────┘
      │
      ▼  (1) ingest.py
┌──────────────────────────────────┐
│ Парсинг → разбивка по статьям    │
│ → embeddings (multilingual MiniLM)│
└─────┬─────────────────────────────┘
      ▼
┌──────────────────┐
│  Chroma vector   │  локальная векторная база (./vector_store)
│  store           │
└─────┬────────────┘
      │
┌─────┴───────────────────────────────────┐
│ FastAPI (app/main.py)                   │
│  GET  /         → веб-чат (ui.html)     │
│  POST /ask      → RAG-ответ + источники │
│  POST /admin/upload, /admin/reindex     │
└─────┬───────────────────────────────────┘
      ▼
┌──────────────────────┐
│ LLM (OpenAI/Ollama)  │ ← конфигурируется через env
└──────────────────────┘
```

### Ключевые решения

| Этап | Решение | Почему |
|------|---------|--------|
| Парсинг | python-docx + regex `^Статья N` | Естественная единица казахстанского НПА — статья |
| Разбивка длинных статей | По абзацам, ≤2000 символов | Эмбеддинги остаются семантически фокусными |
| Эмбеддинги | `paraphrase-multilingual-MiniLM-L12-v2` | Малый, быстрый, хорошо понимает русский юридический язык |
| Векторная БД | Chroma (persistent) | Не нужен внешний сервис, всё в одном контейнере |
| LLM | Pluggable: OpenAI / Ollama | Можно работать офлайн (Ollama) или с топовыми моделями (GPT-4) |
| Промпт | Жёсткий system prompt + цитаты | Запрещает выдумывать статьи; всегда указывает источник |

## Запуск

### Вариант A — c OpenAI

```bash
export OPENAI_API_KEY=sk-...
docker compose up --build
```

Откройте http://localhost:8000

### Вариант B — полностью локально (Ollama)

1. В `docker-compose.yml` раскомментируйте сервис `ollama` и переменные `LLM_PROVIDER=ollama`.
2. Запустите:
   ```bash
   docker compose up --build
   docker compose exec ollama ollama pull llama3.1
   ```

### Вариант C — без Docker

```bash
pip install -r requirements.txt
python -m app.ingest                # один раз для построения индекса
export OPENAI_API_KEY=sk-...
uvicorn app.main:app --reload
```

## Использование

### Веб-чат

http://localhost:8000 — страница с полем ввода. Ответы содержат:
- сам ответ;
- раскрывающийся список источников: название закона, статья, фрагмент текста, оценка релевантности.

### REST API

```bash
curl -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -d '{"query": "Какова страховая сумма при обязательном страховании ответственности нотариуса?"}'
```

### Добавить новый закон

1. Через UI/API:
   ```bash
   curl -F "file=@new_law.docx" http://localhost:8000/admin/upload
   curl -X POST http://localhost:8000/admin/reindex
   ```
2. Или просто положите файл в `laws/` и пересоберите образ.

## Расширение базы знаний с adilet.zan.kz

В `app/ingest.py` можно добавить функцию-загрузчик с adilet.zan.kz —
сайт отдаёт законы в HTML, парсится `BeautifulSoup`-ом, дальше тот же
конвейер: текст → разбивка по статьям → эмбеддинги. Структура `iter_chunks`
уже обобщённая, нужно только дописать второй источник.

## Что заложено в промпт (важное)

- Ассистент **обязан** ссылаться на статью и закон.
- Если в загруженных фрагментах нет ответа — **не выдумывает**, а
  предлагает проверить на adilet.zan.kz.
- Температура LLM = 0.1 — минимум креативности, максимум точности.
